# Upgrade nodectl Quick Start Guide

This guide is specifically for upgrading your node to to the latest version of **nodectl** with opinionated sensible defaults.

## â—½ SSH into Your VPS
Review your [notes](/validate/resources/nodectl-notes) for the connection string.

```
ssh -i /path/to/ssh/private/key nodeadmin@vps_ip_address
```
Refer to [SSH Explanation](/validate/validator/ssh-keys), [Mac SSH Guide](/validate/resources/accessMac), and [Windows SSH Guide](/validate/resources/accessWin)
for detailed understanding.


## â—½ Begin upgrade process
```
sudo nodectl upgrade_nodectl
```

## â—½ Confirm upgrade request
nodectl should offer you `2` versions to choose from.  The latest version of nodectl and the last known upgrade path version.

:::caution UPGRADE PATH
Make sure you use the proper [upgrade path](/validate/automated/nodectl-upgrade-path), upgrading to the latest while ignoring the upgrade path, may create undesired effects.
:::

We will press key ( no need to hit <kbd>enter</kbd> ) the number option related to the verson that best suit our needs and follows the [upgrade path](/validate/automated/nodectl-upgrade-path) accordingly.

Press <kbd>y</kbd> to confirm.

## â—½ Confirm Authentic Release
After the newest version of nodectl is downloaded make sure your receive the message.

**`SUCCESS - AUTHENTIC NODECTL`** 

We can continue the process.

:::danger INVALID SIGNATURE - WARNING
Before using any other nodectl commands, refer to Constellation Network's official Discord channel for assistance. Avoid issuing additional commands to prevent potential interference from nefarious actors impersonating nodectl's functionality.
:::
