# Create SSH Key Pair Quick Start

This guide is specifically for creating a [SSH Key Pair](/validate/validator/ssh-keys) to connect remotely to a Constellation Network node. It is a general guide.

## ◽ Learn SSH Keys (optional)
Detailed step-by-step guides can be found [here](/validate/validator/ssh-keys).

## ◽ Prepare to Create Key Pair
We need to open a **command prompt** or **terminal** window.

<Tabs>
<TabItem value="Choose" label="Choose">
Please choose <b>Windows</b> or <b>Macintosh</b> platform.
</TabItem>

<TabItem value="Windows" label="Windows">

#### ◽ Open a Command Prompt

From the Microsoft Windows start menu type in `Command Prompt` on the main search bar and select the `Command Prompt` icon.

In the search bar at the top, enter in `Terminal`

## ◽ Create SSH Key Pair

<Tabs>
<TabItem value="Choose" label="Choose">
Please choose <b>Windows</b> or <b>Macintosh</b> platform.
</TabItem>

<TabItem value="Windows" label="Windows">

Change to the `.ssh` directory to simplify creation and future locating of your SSH key pairs.
```
cd .ssh
```
Generatate SSH key.
```
ssh-keygen -b 4096 -C ConstellationNetwork
```

<WinCommandPrompt>
C:\Users\myuser\.ssh>ssh-keygen -b 4096 -C ConstellationNetwork<br />
Generating public/private rsa key pair.<br />
Enter file in which to save the key (C:\Users\myuser/.ssh/id_rsa):<br />
</WinCommandPrompt>

Choose a `name` for your key pair (*use whatever name you would like.*)
```
ConstellationNodeKeyPair
```
Enter a complex strong passphrase and confirm the passphrase.

<WinCommandPrompt>
Enter passphrase (emtpy for no passphrase)<br />
Enter same passphrase again:<br />
Your identification has been saved in ConstellationNodeKeyPair.pub<br />
The key fingerprint is:
SHA256: 13vaB0xApJUD13F+Lg9e1uA13rmXaticP0vQd13LBXNk<br />
The key's randomart image is:<br />
+---[RSA 4096]----+<br />
(image code omitted)<br />
+---[SHA256]------+<br />
C:\Users\myuser\.ssh>
</WinCommandPrompt>

We are in the `.ssh` directory off the root of the user's home directory.  

We can do a directory listing to review our newly created key pair.

## ◽ Update Notes
Create a [notes](/validate/resources/nodectl-notes) document or update your notes documentation with your new information:
- public key name
- private key name
- local location for your keys
- SSH key passphrase

## ◽ Creation Complete!
You should now have your SSH private and public keys properly created and ready for use in your `.ssh` subdirectory under your local computer's user account.
